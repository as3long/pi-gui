# 天气查询扩展 (Weather Query Extension) - 改进版

## 📋 概述
这是一个为 pi-gui 创建的天气查询扩展，让用户可以通过 LLM 或命令直接查询全球城市的天气信息。**改进版**提供了更清晰、无ANSI颜色代码的格式化输出。

## 🎯 主要功能

### 1. 自定义工具: `weather_query`
- **用途**: LLM 可以自动调用此工具查询天气
- **参数**:
  - `city` (必需): 城市名称，使用英文或拼音（如 Beijing, Shanghai, Ezhou）
  - `forecast` (可选): 获取详细预报，包括未来几天
  - `brief` (可选): 仅获取当前天气简要信息

### 2. 命令: `/weather`
- **用法**: `/weather <城市名>`
- **示例**: `/weather Beijing`
- **功能**: 直接查询指定城市的天气信息

## 📁 文件位置
```
.pi/extensions/weather-query.ts
```

## 🚀 使用方法

### 方法 1: 通过 LLM 对话
直接告诉 LLM:
> "查询北京今天的天气"
> "上海天气怎么样？"
> "武汉未来三天天气预报"

LLM 会自动识别并调用 `weather_query` 工具。

### 方法 2: 通过命令
在 pi 中直接使用命令:
```
/weather Beijing
/weather Shanghai --forecast
/weather Guangzhou --brief
```

## 🌡️ 天气数据
- **数据来源**: wttr.in 天气服务
- **更新频率**: 实时更新
- **支持范围**: 全球所有城市
- **语言支持**: 中文输出

## ⚙️ 改进特性

### 1. 清晰的输出格式
- **移除ANSI颜色代码**: 输出干净，无乱码
- **结构化布局**: 使用清晰的标题和分隔线
- **天气图标**: 根据天气条件自动添加相应图标
- **中文友好**: 所有描述使用中文

### 2. 三种输出模式
- **简洁模式** (`brief=true`): 单行当前天气信息
- **默认模式**: 当前天气 + 今日预报
- **详细预报模式** (`forecast=true`): 包含未来多天的详细预报

### 3. 错误处理
- 友好的错误消息
- 网络连接检查
- 城市名称验证

## 🔧 技术实现
- 使用 TypeScript 编写
- 基于 wttr.in API 获取天气数据
- 使用正则表达式清除ANSI颜色代码
- 智能天气图标匹配
- 结构化数据格式化

## 🎨 界面集成
- 在状态栏显示天气查询状态

## 🔒 安全性
- 不执行任何系统命令
- 仅通过 HTTPS 调用外部 API
- 不存储任何用户数据
- 网络请求使用标准 fetch API

## 🐛 故障排除
1. **无法获取天气**: 检查网络连接和城市名称拼写
2. **扩展未加载**: 使用 `/reload` 命令重新加载扩展
3. **编码问题**: 已清除ANSI代码，现在兼容所有终端
